## ACTION REQUIRED - deployed contract (UUPS proxy, owner = Safe multisig)

These are **not** auto-edited: changing `.sol` source does nothing on-chain. Each
needs a reviewed implementation + `upgradeToAndCall` via the Safe. Ship together.

1. **HIGH - stuck `Committed` arena strands all stakes.** A full arena moves
   `Created -> Committed`; from there the only exit is `callNumber` by the
   `turnIndex==0` player. If that player never calls, the arena can never be
   cancelled (`cancelArena` requires `Created`) or settled - every stake is locked
   forever. **Fix:** allow `cancelArena` from `Committed` once
   `block.timestamp > createdAt + JOIN_WINDOW` (reuse the existing refund loop;
   widen the state check to `{Created, Committed}` with the timestamp gate).
2. **MED (gas) - `LineLib.lineMasks()` rebuilt every call.** `countCompletedLines`
   reallocates a 12-entry memory array of compile-time constants ~150x per settle.
   **Fix:** make the 12 masks `uint32 constant`s and unroll the 12 checks; delete
   `lineMasks()` and the loop.
3. **LOW - `settle()` is `whenNotPaused` but `withdraw()` is not.** Pausing traps
   unsettled prize pools, partly defeating the "withdrawals stay open" guarantee.
   Decide intent: drop `whenNotPaused` from `settle` (it only credits internal
   balances) or update the `pause()` NatSpec.
4. **LOW - `claimBingo` early-freeze grief.** Any joined player can freeze the call
   sequence at any time. A false claimer can't win (winner is replay-verified), but
   it ends the game early. Document as intended, or gate behind a min `callCount`.
5. **LOW - dead storage** (`CoreStorage.earnings`, `Arena.gasReserve`): never
   read/written. Do NOT remove in-place (storage layout) - drop them from the next
   clean (non-upgrade) deployment only.

## CRITICAL referral sybil farm - residual risk

`POST /api/referral` SIWE-authenticates the *referree*, not a genuine invite, so a
wallet-farm can credit itself the per-referree reward. The on-chain payout is
manual (`/referrals/mark-paid`), which is the real guard - **every payable row must
be human-reviewed before paying.** A proper fix is referrer-authorized invite
tokens (referrer signs to mint a one-time code the referree redeems). Tracked, not
yet built. The code now documents this at the SIWE check.
